핵심 기능 모듈

  • 공지 등록/수정: 유형, 대상, 일정, 해시태그, 첨부
  • 승인 관리: 승인/반려 처리 및 이력 기록
  • 발송 계획: 즉시/정기/지정 시간 스케줄
  • 발송 결과: 성공/실패 및 재시도 기준

연동 & 자동화

  • 메일 발송 API: POST /v1/mail/{sender}
  • Graph 캘린더 API: POST /users/{mailboxEmail}/events
  • 정기 발송 시간: 09:00 / 13:00 / 17:00
  • 스케줄러 기반 묶음 발송 + 즉시 발송 지원

포함 기능

  • 공지 등록/수정 및 승인 관리
  • 발송 계획(즉시/예약) 관리
  • 메일 발송 및 Outlook 캘린더 등록
  • 발송 결과 및 이력 조회
  • 대시보드 제공

제외 기능

  • 모바일 앱/모바일 전용 기능 구축
  • 타 메신저는 연계 가이드만 제공 (실제 발송 구현 범위 아님)
  • 외부 시스템 운영/관리 기능
표준 프로세스 기반 워크플로우 + 스케줄러 발송 + 멀티 채널 확장 구조를 동시에 갖춘 설계

5. 화면별 핵심 기능

화면 설계서 기준

Dashboard

  • 결재대기/발송예정/실패/완료 통계 카드
  • 캘린더 월/주/일 뷰 전환 + 일정 클릭 상세
  • 시스템 점검 일정 리스트(승인된 공지)
  • 공지 유형 통계(도넛) · 부서별 수신 현황(바)
  • 최근 공지 목록 + 점검 완료 공지 등록 모달

공지 등록

  • 공지 유형 선택(일반/점검/보안/인프라)
  • 완료 공지 모드(제목/유형/대상 자동 설정)
  • 수신 대상 선택(서비스/법인/부서) + 법인 전체 발송
  • 공지 내용 입력 + 해시태그 등록
  • 발송일시(오전/오후 정기, 시간 직접, 즉시)
  • Outlook 일정 등록(선택, 날짜/시간)

공지 승인

  • 승인/반려 처리(관리자만 가능)
  • 반려 사유 입력 및 상세 모달
  • 관리자 위임 설정 모달

발송 History

  • 법인/상태/기간/검색어 필터
  • 발송 로그 테이블
  • 실패 건 재발송 요청 버튼
  • 상태/중요도/대상/반려 사유 확인
화면 설계서 4종 기준 핵심 기능만 요약
