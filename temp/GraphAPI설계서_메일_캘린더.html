메일/캘린더 API 인터페이스 설계서

MailUtils 및 OutlookCalendarService 기준 구현 내용을 정리했습니다.

개요

메일 발송 API
MailUtils에서 사용하는 내부 message API.
캘린더 생성 API
OutlookCalendarService에서 사용하는 Microsoft Graph API.
토큰 발급
OAuthUtils를 통한 OAuth2 client credentials 방식.

인증

토큰 발급은 OAuth2 client credentials 방식입니다:

POST {tokenUrl}
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
client_id={clientId}
client_secret={clientSecret}
scope={scope}

아래 API 호출은 모두 Authorization: Bearer {access_token}가 필요합니다.

메일 발송 API message-api

엔드포인트

POST {baseUrl}/v1/mail/{sender}

헤더

  • Authorization: Bearer {access_token}
  • Accept: application/json
  • Content-Type: application/json

요청 바디

{
  "from": "sender@company.com",
  "to": "a@company.com;b@company.com",
  "cc": "c@company.com",
  "bcc": "d@company.com",
  "title": "Subject",
  "content": "...",
  "attachments": [
    {
      "name": "file.pdf",
      "contentBytes": "BASE64_ENCODED_BYTES"
    }
  ]
}

메모

  • to/cc/bcc는 현재 구현에서 세미콜론 구분 문자열입니다.
  • attachments는 선택이며, 포함 시 base64 인코딩이 필요합니다.

캘린더 생성 API graph-api

엔드포인트

POST https://graph.microsoft.com/v1.0/users/{mailboxEmail}/events

헤더

  • Authorization: Bearer {access_token}
  • Content-Type: application/json

요청 바디

{
  "subject": "Notice Title",
  "body": {
    "contentType": "HTML",
    "content": "..."
  },
  "start": {
    "dateTime": "2026-01-22T10:00:00",
    "timeZone": "Asia/Seoul"
  },
  "end": {
    "dateTime": "2026-01-22T11:00:00",
    "timeZone": "Asia/Seoul"
  },
  "attendees": [
    {
      "emailAddress": {
        "address": "user@company.com"
      },
      "type": "required"
    }
  ],
  "isReminderOn": true,
  "reminderMinutesBeforeStart": 15
}

응답

{
  "id": "EVENT_ID",
  "...": "other Graph fields"
}

설정 참고

  • spring.security.oauth2.provider.message (baseUrl, tokenUrl, scope)
  • spring.security.oauth2.provider.microsoft (tokenUrl, scope, tenant)
  • spring.security.oauth2.client (client_id, client_secret)
