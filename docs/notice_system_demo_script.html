공지관리 시스템 데모 스크립트

버전: 2026-01-28 · 대상: PM 10명 · 시간: 12~15분
목표
  • 공지 등록 → 승인 → 발송 → 이력/통계 확인까지 전체 흐름 이해
  • 점검/장애 공지의 완료 공지 생성 흐름 강조
  • Outlook 캘린더 연동, 재발송/재생성 등 운영 편의 기능 소개

시연 흐름

  1. 대시보드에서 전체 현황 파악
  2. 공지 등록 (시스템 점검/장애 공지)
  3. 결재 화면에서 승인/반려 처리
  4. 발송 이력에서 검색/재발송/상세 확인
  5. 캘린더 연동 확인
  6. 완료 공지 등록 흐름

발표 대본(요약)

1) 대시보드

  • 승인대기/승인완료/발송완료/실패 현황 요약
  • 월간 캘린더로 승인/발송 예정 공지 확인
  • 공지 유형 통계로 운영 트렌드 파악
  • 시스템 점검 일정에서 점검/완료 공지 확인

2) 공지 등록

  • 유형/중요도/영향 서비스/수신 대상/내용/일정 입력
  • Outlook 캘린더 등록 선택 가능
  • 점검/장애 공지는 완료 공지 등록 가능

3) 공지 발송 결재

  • 관리자는 승인/반려 처리
  • 일반 사용자는 본인 공지 중심으로 확인
  • 관리자 위임으로 결재 권한 임시 위임

4) 공지 발송 History

  • 상태/기간/검색어/수신부서/작성자로 필터링
  • 상세 보기에서 대상/내용/발송 정보 확인
  • 실패 건 재발송 및 캘린더 재생성 가능

5) 완료 공지 흐름

  • 점검/장애 공지는 완료 공지 버튼으로 즉시 작성
  • 제목/유형/수신 대상 자동 세팅, 내용만 입력
FAQ
  • 일반 사용자 기능: 공지 등록 및 본인 이력 조회
  • 캘린더 위치: NoticeCalendar 전용 캘린더
  • 실패 공지 대응: 재발송 기능 지원
