공지관리 시스템 데모 스크립트 (PM 대상, 10명 기준)
버전: 2026-01-28
시간: 12~15분

목표
- 공지 등록 → 승인 → 발송 → 이력/통계 확인까지 전 과정을 한 번에 이해
- 시스템 점검/장애 공지의 완료 공지 생성 흐름 강조
- Outlook 캘린더 연동, 재발송/재생성 등 운영 편의 기능 확인

환경 준비
- 관리자 계정 1개, 일반 계정 1개
- 데모용 공지 데이터(점검/장애/보안/일반 공지 2~3건)
- Outlook 캘린더(NoticeCalendar 확인)
- 화면: Dashboard → 공지 등록 → 공지 발송 결재 → 공지 발송 History

데모 흐름 요약 (스토리)
1) 대시보드에서 전체 현황 파악
2) 공지 등록 (시스템 점검/장애 공지)
3) 결재 화면에서 승인/반려 처리
4) 발송 이력에서 검색/재발송/상세 확인
5) 캘린더 연동 확인
6) 완료 공지 등록 흐름

발표 스크립트 (대본형)

0. 인사 및 개요 (1분)
- "오늘은 공지관리 시스템의 전체 흐름을 빠르게 보여드리겠습니다."
- "공지 등록부터 승인, 발송, 이력, 통계까지 실제 운영자가 쓰는 시나리오로 시연합니다."

1. 대시보드(현황) (2분)
- 화면: 대시보드
- "여기는 한눈에 공지 상태를 요약하는 대시보드입니다."
- "상단 카드에서 승인대기, 승인완료, 발송완료, 실패 현황을 확인합니다."
- "월간 캘린더에는 승인/발송 예정 공지가 표시됩니다."
- "공지 유형 통계는 실제 공지 유형별로 집계되어 운영 트렌드를 확인할 수 있습니다."
- "시스템 점검 일정은 점검/완료 공지를 한데 모아 보여줘 운영 일정 확인이 쉽습니다."

2. 공지 등록 (3분)
- 화면: 공지 등록
- "일반 공지, 시스템 점검, 시스템 장애, 보안 공지 등 유형별 등록이 가능합니다."
- "필수 항목: 공지 유형, 중요도, 영향 서비스, 수신 대상, 내용, 발송 일정."
- "Outlook 일정 등록 옵션을 켜면 캘린더에 자동 등록됩니다."
- "완료 공지 등록 버튼을 눌러 점검/장애 공지에 대한 완료 공지를 생성할 수 있습니다."

시연 포인트:
- 시스템 점검 공지로 등록
- 발송 일정 선택
- Outlook 등록 체크
- 등록 완료

3. 공지 발송 결재 (2~3분)
- 화면: 공지 발송 결재
- "승인 요청 탭에서 승인 대기 공지를 확인합니다."
- "관리자는 승인/반려 처리 가능, 일반 사용자는 본인이 올린 공지만 볼 수 있습니다."
- "관리자 위임 기능으로 승인 권한을 임시 위임할 수 있습니다."

시연 포인트:
- 관리자 계정으로 승인
- 승인 완료 메시지 확인

4. 발송 History (2분)
- 화면: 공지 발송 History
- "발송 이력은 상태, 기간, 검색어로 필터링 가능합니다."
- "수신부서/작성자 검색도 가능하여 운영 이슈를 빠르게 추적할 수 있습니다."
- "상세 보기에서 수신 대상, 공지 내용, 발송 정보 확인."
- "실패 건은 재발송, 캘린더는 재생성 버튼으로 즉시 조치 가능합니다."

시연 포인트:
- 수신부서/작성자 검색
- 상세 보기 열기
- 재발송 버튼 확인

5. Outlook 캘린더 연동 (1분)
- "등록한 공지는 NoticeCalendar 캘린더로 들어갑니다."
- "캘린더를 켜면 운영 일정이 자동으로 확인됩니다."

6. 완료 공지 흐름 (2분)
- "점검/장애 공지는 완료 공지 작성 버튼으로 바로 후속 공지를 작성합니다."
- "완료 공지는 제목, 유형, 수신 대상이 자동으로 세팅되고 내용만 채우면 됩니다."

마무리 (1분)
- "요약하면, 공지는 등록부터 승인, 발송, 이력/통계, 캘린더 연동까지 한 화면 흐름으로 관리됩니다."
- "운영자는 상태 추적과 후속 조치가 빠르고, 관리자 권한도 위임 관리 가능합니다."
- "질문 받겠습니다."

FAQ(예상 질문)
Q1. 일반 사용자는 무엇을 할 수 있나요?
- 본인 공지 등록 및 본인 공지 이력 조회.

Q2. 캘린더는 기본 캘린더에 들어가나요?
- NoticeCalendar 전용 캘린더에 들어갑니다.

Q3. 완료 공지 등록 조건은?
- 점검/장애 공지이며, 완료 공지가 아직 없을 때 생성 가능합니다.

Q4. 실패 공지 대응?
- 재발송 기능으로 즉시 재처리 가능.

팁(데모 안정성)
- 데모 전 PENDING 공지 1~2건 준비
- 캘린더는 NoticeCalendar 체크 상태 확인
- 승인/반려는 관리자 계정에서 진행
